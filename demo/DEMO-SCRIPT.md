# AgentCo — Demo Script (5-7 минут)
> Дедлайн демо: 21 марта 2026 | Автор: Shadrin (PM)

---

## Intro (30 секунд)

**Говорим:**
> "Представьте: вы ставите цель — и за вас работает целая компания. CEO разбивает задачу, делегирует CTO и PM, те передают дальше разработчикам. Всё происходит автоматически. Это AgentCo."

---

## Шаг 1 — Login (30 секунд)

**Открываем:** `http://[DEPLOY_URL]/auth`

**Показываем:** Экран входа — чистый, минималистичный. Email + password.

**Говорим:**
> "Простой JWT-auth. Регистрируемся за 5 секунд."

*[Вводим email/password, нажимаем Login]*

---

## Шаг 2 — Companies (30 секунд)

**Открываем:** `/` — список компаний

**Показываем:** Список компаний или empty state с CTA "Create your first workspace"

**Говорим:**
> "Компания — это ваша AI-команда. Каждая компания имеет своих агентов, задачи и собственную War Room. Нажимаем на демо-компанию."

*[Кликаем на компанию]*

---

## Шаг 3 — War Room ⭐ WoW-момент (1.5 минуты)

**Открываем:** `/companies/{id}` — War Room tab (открывается по умолчанию)

**Показываем:**
- Карточки агентов с пульсирующими зелёными точками (статус Running)
- Activity Feed справа — живые сообщения от агентов
- Счётчик стоимости ($0.012 и растёт)
- Breadcrumb: `AgentCo / Demo Corp`

**Говорим:**
> "Вот War Room. Здесь LIVE. Агент CEO сейчас думает — видите пульсирующую точку? Это не заглушка, это реальный WebSocket стрим от LangGraph. Каждое сообщение — токен от LLM. Стоимость растёт в реальном времени."

**WoW-момент:**
> "Нажимаем Run на задаче — и смотрим как агент начинает работать прямо здесь, на экране. Иерархия: CEO видит задачу → делегирует → SWE исполняет. N уровней глубины."

*[Нажимаем ▶ Run на задаче в Kanban → переключаемся на War Room → ждём 10 секунд]*

---

## Шаг 4 — Kanban Board (45 секунд)

**Открываем:** Tab "Board" в компании

**Показываем:**
- Колонки: Backlog → Todo → In Progress → Done
- Карточки с assignee (аватар + имя агента), priority badge, кнопка ▶ Run
- Drag & drop между колонками
- Task Detail Sidebar (кликаем на карточку)

**Говорим:**
> "Это полноценный Kanban. Каждая задача имеет исполнителя-агента, приоритет, дедлайн. Кликаем на задачу — видим полный лог выполнения с timestamps. Это не mock — это реальный execution log от LLM."

---

## Шаг 5 — Agents (45 секунд)

**Открываем:** Agents в sidebar или через навигацию

**Показываем:**
- AgentCard: avatar с хэш-цветом, model badge (gpt-4o), статус-пульсация
- Форма создания агента: system prompt textarea, model dropdown (не free text!)
- Agent History tab

**Говорим:**
> "Каждый агент имеет роль, системный промпт и модель. Dropdown — никакого free text, только реальные провайдеры. Агент помнит прошлые задачи через RAG (sqlite-vec). Вот Memory секция — агент накопил контекст из предыдущих запусков."

---

## Шаг 6 — Agent Library (30 секунд)

**Открываем:** Library в sidebar

**Показываем:**
- Глобальная библиотека агентов
- Fork в другой проект

**Говорим:**
> "Сохранили хорошего агента — делитесь с другими командами. Fork в новый проект за один клик. Это Agent Portfolio — агрегированная история по всем форкам."

---

## Шаг 7 — Pitch Closing (30 секунд)

**Возвращаемся на:** War Room

**Говорим:**
> "Подведём итог: 309 тестов проходят, иерархия N уровней работает, WebSocket стрим live. M0 → M2 закрыты за дни — самим AgentCo. Продукт строит себя. Это и есть proof of concept."
