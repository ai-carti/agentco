# ALEX-SECURITY-AUDIT-001 — Backend: security audit + интеграционная валидность

**Priority:** 🔴 HIGH
**Assign to:** Alex (Backend)
**Date:** 2026-03-21

## Задача

Провести полный аудит бэкенда с точки зрения безопасности и корректности работы с продом.

## Что проверить и зафиксировать тикетами:

### Security
- [ ] Все эндпоинты защищены авторизацией (JWT проверяется везде где нужно)
- [ ] Input validation: нет SQL injection, XSS через данные
- [ ] Secrets не логируются (ENCRYPTION_KEY, API keys)
- [ ] Rate limiting работает корректно
- [ ] CORS настроен строго (только ai-carti.github.io)
- [ ] Пароли хешируются (bcrypt/argon2, не MD5/SHA1)
- [ ] JWT: короткий TTL, refresh механизм или явный logout

### Data validity
- [ ] Foreign key constraints работают
- [ ] Cascading deletes корректны (удаление компании → удаление агентов/задач)
- [ ] Все эндпоинты возвращают правильные HTTP статусы (не везде 200)
- [ ] Trailing slash consistency (405 при неправильном slash — BUG-104)

### Integration с прод-фронтом
- [ ] Каждый эндпоинт который использует фронт — проверить вручную curl
- [ ] Убедиться что VITE_API_URL правильно используется во всех запросах
- [ ] WebSocket соединение стабильно в проде
