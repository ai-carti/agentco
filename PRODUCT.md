# AgentOS — Product Plan

> Платформа оркестрации ролевых AI-агентов. Управляй виртуальной командой как настоящей компанией.
> 
> *Параллельно: ВКР Тимы в Бауманке (Q4 2026 — защита)*

---

## 🎯 Vision

Пользователь задаёт цель → AI-команда иерархически её декомпозирует и выполняет.
Каждый агент — это сотрудник с ролью, памятью, инструментами и LLM под капотом.

---

## 🔑 Core Concepts

| Сущность | Описание |
|----------|----------|
| **Компания** | Рабочее пространство. Один проект — одна компания. |
| **Агент** | Сотрудник: роль + системный промпт + LLM + инструменты + память |
| **Иерархия** | Дерево: CEO → CTO/CPO → Tech Lead/PM → SWE/QA/... |
| **Задача** | Task с ID, статусом, приоритетом, назначенным агентом |
| **Спринт** | Временной контейнер для задач |
| **Память** | Долгосрочная (Qdrant) + короткая (Redis) — уникальна для каждого агента |
| **Портфолио** | История выполненных задач агента → переиспользование |

---

## 🏗️ MVP (Q2 2026)

### Минимальная версия, достаточная для демо ВКР

1. **Gateway** — центральный узел маршрутизации задач между агентами
2. **Agent Runtime** — жизненный цикл агента (LangGraph/CrewAI)
3. **LLM Router** — GPT-4o для верхних уровней, Llama 3 для нижних (экономия 30×)
4. **War Room UI** — Sprint Board: Backlog / In Progress / Done
5. **Конструктор иерархии** — визуально задать структуру команды
6. **Базовая память** — Redis session cache + Qdrant для долгосрочного контекста
7. **Трассировка** — LangSmith для отладки цепочек

**Критерий MVP:** CEO-агент получает цель "Напиши PRD для мобильного приложения", декомпозирует на задачи, делегирует PM и CPO, собирает результат. Весь процесс виден в War Room.

- [ ] История версий задачи (что менял агент)

### Исполнение
- [ ] LLM-роутер по уровню иерархии и стоимости
- [ ] MCP-инструменты: браузер, файлы, код, API-вызовы
- [ ] A2A протокол: межагентное взаимодействие
- [ ] Политика эскалации: когда передать задачу выше

- [ ] Tool Sandbox: изолированное выполнение инструментов

### Память и знания
- [ ] Долгосрочная память агента (Qdrant vectors)
- [ ] Embeddings cache (снижение LLM calls на 60%)
- [ ] Shared knowledge base для команды
- [ ] Импорт документов в контекст проекта

### Наблюдаемость
- [ ] Live мониторинг: какой агент что делает сейчас
- [ ] Cost tracker: сколько потрачено на LLM API
- [ ] LangSmith/Langfuse трассировка
- [ ] Метрики: TCR, эскалация, латентность
- [ ] Алерты при аномалиях (loop, cost spike)

### Провайдеры (plug-in архитектура)
- [ ] Интерфейс `IAgentProvider`: модель / OpenClaw / кастомный
- [ ] Поддержка: OpenAI, Anthropic, Ollama, Groq, кастомный endpoint
- [ ] Токен-аутентификация для каждого провайдера

## 🏛️ Архитектура

```
User → Frontend (Next.js 14)
           ↓ REST/WebSocket
       GATEWAY (Node.js 22)
      /    |    |    \
  Agent  Memory  Tool  LLM
 Runtime  Store  Box  Router
(LangGraph) (Qdrant) (MCP) (GPT4o/Claude/Llama)
      \    |    |    /
       Observability
     (LangSmith + Prom)
```

### Ключевые метрики цели
| Метрика | Target |
|---------|--------|
| Latency p50 | < 0.8s |
| Latency p99 | < 3.5s |
| Parallel sessions | 500+ |
| Task Completion Rate | > 87% |
| Time vs manual team | −62% |
| LLM cost efficiency | 3.2× |
| Escalation rate | < 8% |

## 💡 Post-MVP Ideas (from Тима)

### Task Manager Integrations
- **Идея**: задачи агентов синхронизируются с внешними трекерами
- **Примеры**: Linear, Jira, Я.Трекер, Notion, GitHub Issues
- **Механика**: двусторонняя синхронизация — агент создаёт задачу → она появляется в Linear; или задача из Linear назначается агенту
- **Ценность**: AgentCo становится исполнителем в существующем рабочем процессе команды, не требует замены инструментов
- **Приоритет**: post-MVP, после M3
