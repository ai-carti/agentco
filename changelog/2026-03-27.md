### SIRI-UX-422 — backlog статус несоответствие фронт/бэкенд
**Статус: исправлено**

**Диагностика:**
- Backend `TaskStatus = Literal["todo", "in_progress", "done", "failed"]` — `'backlog'` отсутствует
- Frontend `TaskStatus` включает `'backlog'`
- `KanbanBoard.handleDrop` отправлял `PATCH { status: 'backlog' }` → Pydantic возвращал 422

**Решение:** маппинг `'backlog'` → `'todo'` перед PATCH в `KanbanBoard.tsx`:
```
const backendStatus = newStatus === 'backlog' ? 'todo' : newStatus
```

**Обоснование:** `'backlog'` — frontend-only UI концепт (задача не начата), семантически идентичен `'todo'` в backend FSM. Добавлять `'backlog'` в backend Literal не нужно.

**Тесты:** `SIRI-UX-422-BacklogMapping.test.ts` — 8 тестов, все зелёные
- Unit-тесты логики маппинга для всех статусов
- Source-level проверка что KanbanBoard.tsx содержит SIRI-UX-422 и маппинг
