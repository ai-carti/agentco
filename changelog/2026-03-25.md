## Выполнено Siri [05:12 MSK]

### SIRI-UX-298 — `useWarRoomSocket` — лишний `events` state вызывал ре-рендер на каждое WS-сообщение → fixed
- **Проблема**: хук хранил `useState<WsEvent[]>()` и вызывал `setEvents()` при каждом входящем сообщении. Ни один компонент не читал `.events` — `WarRoomPage` деструктурировал только `{ isConnected, error }`. Каждое WS-сообщение = лишний ре-рендер.
- **Fix**: удалён `events` state и `setEvents()` из `useWarRoomSocket.ts`. Интерфейс `UseWarRoomSocketResult` обновлён — убрано поле `events`.
- **Сопутствующие изменения**: обновлены `useWarRoomSocket.test.tsx` (2 теста переписаны под проверку через store) и `useWarRoomSocket.integration.test.ts` (5 тестов — проверка событий теперь через `warRoomStore.getState()`).
- **Тест добавлен**: `SIRI-UX-298-NoEventsState.test.ts` (4 теста: нет `useState<WsEvent[]>`, нет `setEvents(`, return не содержит `events`, интерфейс не содержит `events: WsEvent[]`)

### SIRI-UX-299 — `KanbanBoard` — `handleDragLeave` без `relatedTarget` — синяя рамка мигает при перетаскивании → fixed
- **Проблема**: при перетаскивании карточки в колонку браузер стреляет `dragleave` на колонке каждый раз, когда указатель проходит над дочерним `TaskCard`-элементом → `setDragOverCol(null)` → синяя рамка колонки мигала.
- **Fix**: `handleDragLeave` теперь принимает `e: React.DragEvent` и проверяет `e.currentTarget.contains(e.relatedTarget as Node)` — если следующая цель внутри колонки, ранний `return` без сброса.
- **Тест добавлен**: `SIRI-UX-299-DragLeaveBorder.test.ts` (3 теста: callback принимает DragEvent параметр, использует `relatedTarget` + `contains()`, имеет `return` guard перед `setDragOverCol(null)`)
